Set Terminology and Notation

set
is a collection of mathematical objects in which each object is consider to occur at most once.
elements
the objects belonging to a set
cardinality
The number of elements in a finite set. We cannot denote the cardinality of an infinite set.
(\(S_1 \subset S_2\))
All of the elements of \(S_1\) appear in \(S_2\). But the elements of \(S_2\) are not necessarily all found in \(S_1\).
(\(S_1 \subseteq S_2\))
All of the elements of \(S_1\) appear in \(S_2\) and all of the elements of \(S_2\) appear in \(S_1\). The two sets are said to be equal.

A set is a collection of objects in which each object is considered to occur at most once. The objects belonging to a set are its elements. We use curly braces \(\{1, 2, 3, 4\}\) to indicate a set. The order in which the elements are listed is not significant. The symbol \(\in\) is used to indicate that an object belongs to a set.

One set \(S_1\) is contained in another set \(S_2\) written (\(S_1 \subset S_2\)) if every element of \(S_1\) belongs to \(S_2\). Two sets are equal if they contain exactly the same elements.

A set can be infinite. If a set if not infinite we use \(|S|\) to denote it’s cardinality, the number of elements it contains. For example the set of suits in a card deck has cardinality 4.

Cartesian product

The *Cartesian production of two sets \(A\) and \(B\) is the set of all pairs \((a,b)\) where \(a \in A\) and \(b \in B\).

For finite sets \(A\) and \(B\) the cardinality of the Cartesian product of \(A\) and \(B\) (written \(|A \times B|\) ) is \(|A| \cdot |B|\).

###The Function

function
(informal) a rule that for each element in some set \(D\) of possible inputs assigns a possible output.
(formal) a (possibly infinite) set of pairs \((a, b)\) no two of which share the same first entry.

Example
The double function with domain \(\{1,2,3,...\}\) is
\[\{(1,2), (2,4), (3,6), ...\}\]

image
The output is said to be the image of the input under the function.
The image of a function \(F\) is the set of images of all domain elements. That is, the image of \(f\) is the set of elements of the co-domain that actually occur as outputs. For example the image of the cosine function is the set of real numbers between 1 and -1.
pre-image
The input is a pre-image of the output.
domain
The set \(D\) of possible inputs is called the domain of the function.
maps to
For a function named \(f\) the image of \(q\) under \(f\) is denoted by \(f(q)\). If \(r = f(q)\) we say that \(q\) maps to \(r\) under \(f\). The notation for “q maps to r” is \(q \mapsto r\).
co-domain
The co-domain is a set from which the function’s output values are chosen. Note that one has some leeway in choosing the co-domain since not all of its members need to be outputs. The notation \(f : D \mapsto F\) means that \(f\) is a function whose domain is the set \(D\) and whose \(co-domain\) (the set of all possible outputs is the set \(F\). (Succinctly, a function that maps \(D\) to \(F\)).

Functions vs. Procedures vs. Computational Problems

Procedure
A precise description of a computation. It accepts inputs (called arguments) and produces an output (called the return value).
Computation problem
An input-output specification that a procedure might be required to satisfy.

Example

  • input: a pair (p,q) of integers greater than 1
  • output: the product pq
  • How is a computational problem different from a procedure?
    • A procedure is basically a specific implementation used to solve a computational problem. One computational problem is integer multiplication. There are many different procedures you could use to solve this problem, some better than others.
    • Unlike a function a computation problem need not specify a unique output for every input.

The two computational problems related to a function

The forward problem
Given an element \(a\) of \(f\)'s domain, compute \(f(a)\) the image of \(a\) under \(f\).
The backward problem
Given an element \(r\) of the co-domain of the function, compute any pre-image (or report that none exists).
  • Solving the backwards problem
    • The only universal way to figure out which input yielded a certain output is to iterate through all the inputs, figuring out what each input maps to, and then stopping when you reach the desired output. This is not fast.
With great generality comes great computational difficulty
By limiting ourselves to a specific family of functions, in our case linear functions we reduce the generality and therefore the computational difficulty, and can solve the backward problem.

Notation for the set of functions with given domain and co-domain

\(F^D\)
all the functions from \(D\) to \(F\).
For any finite sets \(D\) and \(F\), \(|D^F| = |D|^{|F|}\)

####Identity Function

Identity Function
For any domain \(D\) there is a function \(id_D : D \mapsto D\), defined \(id_D(d) = d\)

Composition of functions

Functional composition
An operation that combines two functions to get a new function. Given two functions \(F : A \mapsto B\) and \(g : B \mapsto C\) the function \(g \circ f\) is called the composition of \(g\) and \(f\). It is a function whose domain is \(A\) and whose co-domain is \(C\). It is defined by
\[(g \circ f)(x) = g(f(x))\]

If the image of \(f\) is not contained in the domain of \(g\) then \(g \circ f\) is not a legal expression.

Associativity of function composition

Associativity of composition
For functions \(f, g, h\) \[h \circ (g \circ f) = (h \circ g) \circ f\] if the compositions are legal. This means that parentheses are unnecessary in function composition.

Functional Inverse

Functional Inverse
We say that functions \(f\) and \(g\) are functional inverses of each other if
  • \(f \circ g\) is defined and is the identity function on the domain of \(g\) and
  • \(g \circ f\) is defined and is the identity function on the domain of \(f\)
Invertible
Not every function has an inverse. A function that has an inverse is said to be invertible.
One-to-One
A function \(f\) is one-to-one if for every \(x, y \in D, f(x) = f(y) \implies x = y\)
Onto
a function \(f\) is onto if for every \(z \in F \, \exists \, x \in D | f(x) = z\)
Invertibility Theorem
A function is invertible if and only if it is one-to-one and onto.
Every function has at most one functional inverse.

Invertibility of the composition of invertible functions

If \(f\) and \(g\) are invertible functions and \(f \circ g\) exists then \(f \circ g\) is invertible and \((f \circ g)^{-1} = g^{-1} \circ f^{-1}\)

Probability

Uniform Distributions

All outcomes are equally likely so they are all assigned the same probabilities.

Nonuniform distributions

All outcomes are not equally likely so they are assigned different probabilities.

Events and adding probabilities

Fundamental Principle of Probability Theory
The probability of an event is the sum of the probabilities of the outcomes which make up the event.

Applying a function to a random input

Given a set of random inputs \(\{1,2,3,4,5,6\}\) for example. Count how many of the inputs yield a certain output. Use the sum to define the probability.

If a function is invertible the probabilities are preserved. The probabilities of various outputs match the probabilities of the inputs. Thus if the inputs are chosen uniformly at random, then the outputs will be uniformly random.

Perfect Secrecy

A cryptosystem has to satisfy two requirements

  • The intended recipient of an encrypted message must be able to decrypt it and
  • someone for whom the message was not intended should not be able to decrypt it.

The Kerckhoffs Doctrine states that the security of a cryptosystem should depend only on the secrecy of the key used, not on the secrecy of the system itself.

Beyond not just decrypting the message, the method of encryption should not yield probabilistic likelihoods of certain messages.

Perfect Secrecy and Invertiblity Functions

If functions are invertible then the input and output have the same distribution, so knowing the output yields no information about the input. We say that this scheme yields perfect secrecy.
